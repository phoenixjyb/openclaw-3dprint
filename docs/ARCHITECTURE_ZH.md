# OpenClaw 3D Print — 系统架构文档

## 1. 概述

OpenClaw 3D Print 是一套端到端的自动化管道，能够将自然语言聊天消息转化为实体 3D 打印物件。系统从多种输入渠道（Telegram、飞书、OpenClaw Agent HTTP API）接收文本，可选地通过大语言模型（LLM）对提示词进行丰富优化，随后调用云端 API 生成 3D 网格模型，在本地或远程完成切片，最终将文件发送到局域网内的 Bambu Lab 打印机进行打印。

**核心流程：** 聊天输入 → LLM 提示词优化（可选） → 3D 网格生成 → 切片 → 上传打印机 → 实体物件

## 3. 组件依赖矩阵

| 组件 | 是否必需？ | 用途 | 可替代方案 |
|------|-----------|------|-----------|
| Python ≥ 3.11 | ✅ 必需 | 运行环境 | — |
| Tripo3D API 密钥 | ✅ 必需 | 3D 网格生成 | Meshy.ai、Magic3D、Neural4D、Rodin AI |
| Bambu 打印机 + 局域网访问 | ✅ 必需 | 实体打印 | — |
| PrusaSlicer | ✅ 必需（本地模式） | 将网格切片为可打印的 .3mf 文件 | OrcaSlicer，或远程 Bambu Studio |
| LLM API 密钥 | ❌ 可选 | 提示词丰富优化，提升网格生成质量 | OpenClaw Agent 自带 LLM，或手动编写提示词 |
| Telegram Bot | ❌ 可选 | 聊天输入渠道 | 飞书、OpenClaw Agent HTTP API |
| 飞书 Bot | ❌ 可选 | 聊天输入渠道 | Telegram、OpenClaw Agent HTTP API |
| Windows PC | ❌ 可选 | 通过 Bambu Studio 远程切片 | 本地 PrusaSlicer (Mac/Linux) |
| paramiko (SSH) | ❌ 可选 | 与远程 Windows 通信 | 使用本地切片时无需此组件 |

## 4. 管道路线

系统支持多种管道路线，可根据实际环境和需求灵活选择。

### 路线 A：OpenClaw Agent（推荐）

```
OpenClaw Agent → enriched_prompt → HTTP API (端口 8765)
  → Tripo3D 网格生成
  → PrusaSlicer (本地切片)
  → FTPS 上传 + MQTT 打印
```

**所需组件：** Tripo3D API 密钥 + Bambu 打印机 + PrusaSlicer  
**无需 LLM 密钥** — OpenClaw Agent 自身具备 LLM 能力，可自行完成提示词优化。

这是推荐路线，因为 Agent 本身已内置 LLM 能力，能生成针对 3D 打印优化的高质量提示词。

### 路线 B：Telegram 独立模式

```
Telegram 消息 → 内置 LLM 提示词优化
  → Tripo3D 网格生成
  → PrusaSlicer (本地切片)
  → FTPS 上传 + MQTT 打印
```

**所需组件：** Telegram Bot Token + LLM API 密钥 + Tripo3D API 密钥 + Bambu 打印机 + PrusaSlicer

在此模式下，管道自身会调用 LLM 将用户的原始文本优化为适合 3D 打印的提示词。

### 路线 C：远程 Windows 切片

```
与路线 A 或 B 相同，但切片在远程 Windows PC 上完成：
  → SSH 连接到 Windows 机器
  → 使用 Bambu Studio CLI 切片
  → .3mf 文件传回本地
  → FTPS 上传 + MQTT 打印
```

**所需组件：** 路线 A 或 B 的全部组件 + Windows PC + SSH 访问权限 + paramiko

当需要使用 PrusaSlicer 不支持的 Bambu Studio 特有切片功能或打印配置时，可选择此路线。

### 路线 D：纯命令行手动模式

```
终端 → `3dprint request --enriched "一只底座稳固的可爱小猫" description`
  → Tripo3D 网格生成
  → PrusaSlicer (本地切片)
  → FTPS 上传 + MQTT 打印
```

**所需组件：** Tripo3D API 密钥 + Bambu 打印机 + PrusaSlicer  
**无需任何 Bot 或 Agent。** 用户直接提供优化后的提示词即可。

## 5. 支持的打印机

| 打印机 | FTPS (990) | MQTT (8883) | 局域网模式 | 备注 |
|--------|-----------|------------|-----------|------|
| Bambu Lab X1 / X1 Carbon / X1E | ✅ | ✅ | ✅ | 局域网打印需插入 SD 卡 |
| Bambu Lab P1S / P1P | ✅ | ✅ | ✅ | 需开启开发者模式 |
| Bambu Lab A1 / A1 Mini | ✅ | ✅ | ✅ | 入门级，协议相同 |

> **说明：** 任何支持局域网模式并开启了开发者模式的 Bambu Lab 打印机均可使用。打印机必须与运行管道的设备处于同一局域网内。
>
> - **开启局域网模式：** 打印机 → 设置 → 网络 → 仅局域网模式
> - **开启开发者模式：** 打印机 → 设置 → 网络 → 开发者模式

## 7. 回退与错误处理

| 故障场景 | 处理策略 |
|---------|---------|
| **网格生成失败** | 指数退避重试最多 3 次 → 通知用户 → 取消任务 |
| **找不到切片软件** | 检查备选安装路径（Linux 与 Mac 默认路径不同） → 返回清晰的错误信息及安装指引 |
| **打印机无法连接** | FTPS 连接重试 3 次 → 检查 MQTT 连通性 → 通知用户并附上排查步骤 |
| **LLM 不可用（独立模式）** | 返回错误："请配置 `OPENAI_API_KEY`，或直接使用 `enriched_prompt` 参数" |
| **审批超时** | 1 小时超时 → 自动取消任务 → 通知用户 |
| **打印机队列已满** | 显示队列位置 → 等待空闲 → 有空位时通知用户 |

| 端口 | 协议 | 方向 | 用途 |
|------|------|------|------|
| 990 | FTPS (隐式 TLS) | 管道 → 打印机 | 上传 `.3mf` 文件到打印机 |
| 8883 | MQTTS (TLS) | 管道 ↔ 打印机 | 打印指令下发 + 状态监控 |

| 22 | SSH | 管道 → Windows | 远程切片（可选） |

## 9. 安全

- **密钥管理：** 所有 API 密钥、令牌和凭据存储于 `~/.openclaw-3dprint/pipeline.env`，严禁提交到代码仓库。
- **打印机通信：** 与打印机的所有通信均使用 TLS 加密（FTPS 用于文件上传，MQTTS 用于指令传输）。
- **HTTP API 绑定：** API 服务仅监听 `127.0.0.1`（本地回环地址），不对外暴露。
- **Telegram Bot 鉴权：** 通过用户 ID 白名单校验，防止未授权访问。
- **Git 安全：** `.gitignore` 已排除所有 `.env` 文件及 staging 临时目录。

## 10. 打印监控与通知

后台 **PrinterMonitor** 订阅打印机的 MQTT 消息流，对所有打印事件发送实时通知——无论是流水线触发的还是在 Bambu Studio 中手动启动的。

### 监控事件

| 事件 | 触发条件 | 通知内容 |
|------|---------|---------|
| **开始打印** | 状态变为 `RUNNING` | 任务名称、预计时间 |
| **打印进度** | 每 N%（可配置） | 百分比、剩余时间 |
| **打印完成** | `RUNNING` → `FINISH` | 总耗时 |
| **打印失败** | 状态变为 `FAILED` | 错误详情 |
| **打印暂停** | `RUNNING` → `PAUSE` | 暂停原因 |
| **HMS 告警** | 硬件/固件告警 | 告警代码 + 描述 |

### 双通道通知

- **Telegram**（ybcc）— 通过 `python-telegram-bot` + ClashX 代理
- **飞书**（chuan）— 通过飞书开放平台 API + ClashX 代理

两个通道独立运行，任一通道失败不影响另一个。

### macOS 部署：MQTT TLS 代理

在 macOS 上，Homebrew 安装的 Python 因 **本地网络隐私保护（TCC）** 无法访问局域网设备。流水线通过 TLS 终端代理绕过此限制：

```
流水线 (brew Python) ──明文TCP──▶ MQTT 代理 (系统 Python) ──TLS──▶ 打印机:8883
     localhost:18883                /usr/bin/python3
```

**注意：** 代理必须作为独立的 **launchd agent** 运行，不能作为流水线的子进程。macOS 会将父进程的局域网限制继承给子进程。

| 组件 | launchd 标签 | 运行方式 |
|------|-------------|---------|
| 流水线 | `com.openclaw-3dprint.pipeline` | `run-pipeline.zsh` → brew Python 虚拟环境 |
| MQTT 代理 | `com.openclaw-3dprint.mqtt-proxy` | `/usr/bin/python3 scripts/mqtt-proxy.py` |

## 11. 文件流转

```
用户提示词 (文本)
  → enriched_prompt (优化后的提示词文本)
  → model.glb / model.stl       (3D 网格模型, ~5-50 MB)
  → model_sliced.3mf             (切片后的打印文件, ~2-20 MB)
  → 上传到打印机 /cache/         (通过 FTPS)
  → MQTT 打印指令                (启动打印任务)
  → 实体物件
```

所有中间文件存储在 `~/.openclaw-3dprint/staging/` 目录下，打印任务成功完成后自动清理。
